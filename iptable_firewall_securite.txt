1. Protection contre les attaques SYN flood
Limiter le nombre de connexions TCP SYN :


iptables -A INPUT -p tcp --syn -m limit --limit 5/second -j ACCEPT
iptables -A INPUT -p tcp --syn -j DROP
2. Protection contre les attaques Smurf (Ping flood amplifi√©)
Bloquer les pings vers les adresses de diffusion :


iptables -A INPUT -p icmp --icmp-type echo-request -j DROP
3. Protection contre les attaques UDP flood
Limiter les paquets UDP :


iptables -A INPUT -p udp -m limit --limit 10/second -j ACCEPT
iptables -A INPUT -p udp -j DROP
